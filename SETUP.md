# 飞书日程复盘机器人 —— 部署指南

## 你需要准备什么

1. 一个飞书账号（个人即可）
2. 一个 GitHub 账号
3. 一个 Vercel 账号（用 GitHub 注册）
4. 一个 DeepSeek API Key

成本预估：**完全免费**（个人使用量在免费额度内）

---

## 第一步：在飞书创建自建应用

### 1.1 进入飞书开放平台

浏览器打开 https://open.feishu.cn

用你的飞书账号登录。

### 1.2 创建企业

如果没有企业，系统可能会提示让你创建一个"开发者企业"——按照提示操作即可，免费的。

### 1.3 创建自建应用

1. 进入"开发者后台" → "应用" → "创建应用"
2. 填入应用名称（随便起，比如"日程复盘"）
3. 点击"创建"

### 1.4 获取应用凭证

在应用页面左侧找到 **"凭证与基础信息"**，复制：
- **App ID**（`cli_xxx`）
- **App Secret**（点"显示"后复制）

这两个稍后会填入 Vercel 环境变量。

### 1.5 配置机器人

左侧点击 **"应用功能"** → **"机器人"**，点击开启。

### 1.6 配置事件订阅

左侧点击 **"事件订阅"**：

1. 先不要填"请求网址"（等部署后拿到 Vercel 域名再填）
2. 在"事件"列表中搜索 `接收消息`，点击添加
3. 找到 **"接收消息"** 事件 → 选择 **"im.message.receive_v1"**

### 1.7 添加权限

左侧点击 **"权限管理"**，搜索并开通以下权限：
- `im:message`（获取消息）
- `im:message:send`（发送消息）
- `im:image:upload`（上传图片，用于发送日程复盘图表）

---

## 第二步：获取 DeepSeek API Key

1. 打开 https://platform.deepseek.com
2. 注册/登录
3. 进入 "API Keys" → 创建新 Key
4. 复制保存（`sk-xxx`）

费用很低，个人使用一个月几块钱足够。

---

## 第三步：部署到 Vercel

### 3.1 把代码上传到 GitHub

1. 在 GitHub 新建一个仓库（同名 feishu-bot，Private 即可）
2. 把本目录下的所有文件推到这个仓库

### 3.2 连接 Vercel

1. 打开 https://vercel.com，用 GitHub 登录
2. 点击 "New Project" → 选择你的仓库
3. 在 "Framework Preset" 中选择 "Other"

### 3.3 配置环境变量

在 Vercel 项目页面，左侧进入 **Settings → Environment Variables**，依次添加：

| 变量名 | 值 |
|--------|-----|
| `FEISHU_APP_ID` | 第一步复制的 App ID |
| `FEISHU_APP_SECRET` | 第一步复制的 App Secret |
| `FEISHU_USER_ID` | （等第五步查到了再填，先跳过） |
| `DEEPSEEK_API_KEY` | 第二步复制的 DeepSeek Key |
| `CRON_SECRET` | 随便输一串随机字符（保护 cron 接口） |

### 3.4 部署

回到 "Deployments" 标签 → 点击 "Deploy"。

部署成功后你会得到一个域名，类似 `https://feishu-bot-xxx.vercel.app`。

---

## 第四步：完成飞书事件订阅配置

### 4.1 填入回调地址

回到飞书开放平台 → 你的应用 → **"事件订阅"**：
- **请求网址**填入：`https://你的vercel域名.vercel.app/api/feishu`
- 点击"保存"

飞书会立即向这个地址发送验证请求，如果配置正确，验证会通过。

### 4.2 发布应用

左侧点击 **"版本管理与发布"**：
1. 点击"创建版本"
2. 填写版本号（如 1.0.0）
3. 点击"发布"

---

## 第五步：获取你的飞书用户 ID 并测试

### 5.1 在飞书中找到你的 bot

打开飞书客户端 → 搜索栏搜索你的应用名称 → 找到机器人 → 点击进入对话。

### 5.2 获取 open_id

对你刚创建的机器人**发一条消息**（任意内容，比如"测试"）。

在你部署的 Vercel 项目中，进入 **Functions** 标签 → 找到 `feishu` 函数的日志，你会看到类似：
```
收到消息: "测试" from ou_xxxxxxxxxxxx
```

复制这个 `ou_xxx` → 这就是你的 `FEISHU_USER_ID`。

回到 Vercel → Settings → Environment Variables → 填入 `FEISHU_USER_ID` → 重新部署。

### 5.3 测试

在飞书给机器人发一条日程消息，比如："我到教学楼学习数学了"

如果收到回复"收到！已记录日程。"说明配置成功。

---

## 第六步：等待零点推送

每天零点（北京时间），机器人会自动生成前一天的日程复盘图表并推送给你。

---

## 测试命令

在飞书跟机器人对话时，可以使用以下命令：

| 命令 | 功能 |
|------|------|
| `/数据` | 查看今天已存储的所有消息的 JSON 原始数据 |
| `/预览` | 立即生成今天的日程复盘图表（图片），看看当前数据画出来什么效果 |

---

## 常见问题

**Q: 收到消息没有回复？**
检查 Vercel Functions 日志，看看是否有错误。常见原因是 DeepSeek API Key 不对或余额不足。

**Q: 零点没有推送？**
检查 Vercel → Deployments 中 cron 是否运行成功。

**Q: Vercel 免费额度够用吗？**
个人使用完全够：100GB 带宽、100 万次函数调用/月、cron 每天一次。
